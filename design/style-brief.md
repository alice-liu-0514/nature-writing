# 設計風格 Brief：劉芸辰自然書寫作品集

---

## 設計方向總綱

這個網站是一個台灣學生的私人土地回憶錄，三篇散文共享同一片地景——大肚溪出海口旁的田地「過溪」。設計方向應以**「有重量的留白」**為核心：不是精緻的空洞，而是讓讀者感覺到土壤、水氣、風聲存在的那種靜默。字體選明體以召喚紙本閱讀的文學感，配色取自田地與水塘的低彩飽和色，滾動敘事克制使用——只在段落轉折或感官意象強烈的地方介入。整體目標是讓讀者讀完像讀完一本薄薄的散文集，而不像瀏覽過一個「網站」。

## 參考方向

### 1. Craig Mod — craigmod.com/essays/

**為什麼相關：** Craig Mod 是旅行 / 地景散文寫作者，長年思考「如何讓數位長文讀起來像書」。他的每篇 essay 都是沉浸式的單頁閱讀體驗，攝影與文字的關係和本案非常接近——不是圖說，而是圖與文共同呼吸。

**可借用的具體手法：**
- **閱讀欄寬設定**：正文欄限制在約 `max-width: 680px`，並大幅內縮對齊，左右留大量空白——不是置中卡片，而是把欄寬當成「書的版心」來對待。
- **背景非純白**：使用略暖的米白（約 `#faf8f4`）而非 `#ffffff`，降低螢幕的眩光感，給閱讀一種紙張質地。
- **圖片全出血插入**：重要攝影圖片突破文字欄，直接佔滿瀏覽器全寬，形成段落呼吸點，下方立刻回到窄欄文字——節奏感來自「窄→寬→窄」的交替。
- **無滾動動畫**：段落之間不用任何 fade-in 或 slide。「克制」本身就是設計。

### 2. NYT Snow Fall / 衛報 Firestorm 類型的沉浸長文

**代表：** NYT Snow Fall（2012，定義了 scrollytelling 的起點）、The Guardian Firestorm  
**URL：** 直接參考分析文 https://shorthand.com/the-craft/scrollytelling-examples/

**為什麼相關：** 這類作品發展出了「sticky 文字＋背景圖換場」的基本語法，後來的自然書寫長文大量沿用。但其核心不是動畫量，而是**敘事節奏的可視化**——讀者捲動的速度等於閱讀的速度。

**可借用的具體手法：**
- **sticky 文字區塊**：一張全屏圖佔 `position: sticky` 固定在背景，文字段落從底部捲上來覆蓋，到下一個段落才換圖。適合「魚」篇的三層生態觀察——每一層（水面 / 水中 / 水底）各配一張圖，段落捲動帶動圖片切換。
- **章節標題用大字+大量上方留白作為「呼吸停頓」**：章節間距不用分隔線，改用 `padding-top: 25vh` 讓讀者感覺到時間流逝。
- **字體大小的對比層級**：章節標題 `clamp(2.8rem, 5vw, 4.5rem)`，引言句（chapeau）`1.4rem` 斜體，正文 `1.125rem`——三層明確分工，不需要裝飾元素。
- **視差只做一件事**：不要全站視差。只在文章開頭的 hero 圖做 `background-attachment: fixed`，其他地方不用——否則閱讀節奏會被分散。

### 3. Kinfolk Magazine 的排版美學

**為什麼相關：** Kinfolk 處理的題材（日常生活、食物、手工、自然）和本案的感官重心高度重疊。它的印刷排版語言——大量留白、衝突的字級對比、明體（Serif）配細黑——是「自然書寫作品集」在視覺上的最佳對標之一。

**可借用的具體手法：**
- **Serif 標題 + 細 Sans-serif 圖說**：標題用明體（Noto Serif TC）展現文學氣息；圖說（caption）改用細字重的黑體（Noto Sans TC Light），字級縮小到 `0.8rem`，顏色降成 `#888`——兩種質地形成清楚的角色分工。
- **不對稱排版的圖文節奏**：在桌機版，一些段落的圖片靠左或靠右突出欄位，形成版面上的不規則感。不要每張圖都全寬置中——那是通用模板的特徵。
- **大字字距（headline letter-spacing）**：標題字 `letter-spacing: 0.15em`，給中文標題一種平靜、慎重的質感，避免文字堆疊感。
- **留白作為敘事元素**：Kinfolk 的版面密度刻意比「資訊網站」低 40-50%。本案的每個段落後加 `margin-bottom: 2.5rem`，圖片上下各留 `4rem`，讓內容「呼吸」。

### 4. 報導者（The Reporter）— twreporter.org 的長文閱讀系統

**URL：** https://www.twreporter.org  
**為什麼相關：** 台灣在地的嚴肅長文平台，中文閱讀體驗設計扎實，欄寬、行高、字體選擇都有明確的編輯判斷。可以當作「台灣讀者習慣的中文閱讀基準線」。

**可借用的具體手法：**
- **正文欄寬**：約 `600-680px`（約 30-35 個中文字/行），符合中文閱讀最適行長。CSS 實作：`max-width: 42rem; margin: 0 auto; padding: 0 1.5rem;`
- **正文行高 `1.9`**：比英文慣用的 `1.5` 更鬆，因為中文字符密實，需要更多垂直呼吸空間。
- **字體 Stack**：`'Noto Serif TC', 'Source Han Serif TC', serif` 用於正文；標題可加入字重對比（`font-weight: 700`）。
- **深灰而非純黑的文字色**：正文 `#333333` 或 `#2c2c2c`，避免純黑 `#000000` 在暖底上的視覺刺激。

### 5. 台灣設計師王志弘（Wang Zhi-Hong）的書籍設計語言

**URL：** https://wangzhihong.com  
**為什麼相關：** 王志弘是台灣當代書封設計的重要人物，他的設計核心是「字體即圖形」——讓漢字的結構本身成為視覺力量，不依賴裝飾。這個哲學直接適用於本案的章節扉頁設計。

**可借用的具體手法：**
- **篇名作為視覺主角**：三篇篇名（花生、魚、竹）可以用極大字級（`font-size: clamp(5rem, 15vw, 12rem)`）配合明體，作為每篇的 hero 視覺。字本身就是圖像，不需要任何圖示或插畫。
- **漢字筆畫的呼應**：「竹」字的垂直結構可以對應版面的縱向排列；「魚」字可以橫向展開——讓字形的幾何特質影響版面方向。
- **黑白灰的主版面，只用一個強調色**：王志弘的書封通常是低彩度主體加上一個精確的顏色點。本案每篇可以各有一個主調色（見「具體建議」段落），其他元素保持中性。

## 具體建議：三篇各自的設計定調

### 花生 — 詼諧、感官、泥土、踏實

**配色：**
- 主背景：暖米白 `#f7f3ec`（乾燥土壤的顏色）
- 主調色：赤陶橙 `#c4622d`（花生殼的紅棕）
- 文字：深咖啡 `#3a2a1a`
- 輔助：草綠 `#7a8c4a`（田邊的雜草）

**版面氣質：**
段落之間不需要太多戲劇性。這篇的語調是詼諧的，版面也應該踏實、直白——不用 sticky 或視差。圖片採「左右交替突出欄位」的排版（桌機版：`.image-left { margin-left: -6rem; }`），模擬相片夾在日記裡的感覺。田間的泥濘、陷腳的感官細節，可以在關鍵句段落前加一條 `2px` 粗的赤陶色左邊線（`border-left`），讓感官意象視覺化，不用任何動畫。

**互動重點：**
圖片點擊後以 lightbox 方式展開（不要另開新頁），caption 寫在圖片下方而非浮動覆蓋——樸實地放著，就是這篇的格調。

---

### 魚（魚塘）— 靜觀、神秘、水的深度

**配色：**
- 主背景：深石板藍黑 `#1a2530`（夜晚水塘的顏色）
- 主調色：暗青 `#2a7a8a`
- 文字：偏冷的白 `#e8edf0`（在深背景上）
- 輔助：泥黃 `#b89a60`（池底的沉積色）

**版面氣質：**
這是三篇中唯一適合使用深色背景的篇章。文字在深背景上的長文閱讀，行高需要提高到 `2.1`，字重維持在正常（`400`），否則白字在暗底容易發光刺眼。「三層生態」的結構天然適合做 **sticky 圖層切換**：水面（白日光影）→ 水中（模糊混濁）→ 水底（接近黑色的深）。每層圖片用 `filter: brightness()` 或實際換圖，配合文字段落捲動觸發。

**互動重點：**
在「魚的游動」段落可以加一個微小的 CSS 動畫——一個半透明的橢圓形元素在文字欄邊緣緩慢漂移（`animation: drift 8s ease-in-out infinite alternate`）。只做一個，不能更多。這是這篇唯一的「動」的東西。

---

### 竹子（竹林）— 記憶、失去、韌性、傳承

**配色：**
- 主背景：暖灰白 `#f2f0eb`（廢墟竹林的霧感）
- 主調色：竹青 `#4a6741`（老竹的顏色，不是鮮亮的綠）
- 文字：`#2c2c2c`
- 輔助：淡銀灰 `#9a9a97`（折斷的竹子截面）

**版面氣質：**
這篇有時間縱深——颱風前 / 颱風後 / 阿公的記憶。版面可以做最明確的「章節感」：三個時態各自是獨立的版塊，用 `100vh` 高的空白段落隔開，讓讀者捲動時感覺到「時間的跳躍」。阿公相關的段落可以在左邊緣加細線，或正文字體改為略斜的 `font-style: italic`，在語感上區分「回憶的聲音」。

**互動重點：**
竹子的「斷裂」可以對應一個視覺效果：在颱風段落，滾動到特定位置時全版面圖片從正常切換到 `filter: grayscale(80%)`，模擬颱風過後色彩褪去的感覺。用 `IntersectionObserver` 觸發 class 切換，不需要任何 library。

## 要避免的 AI 通用感

以下是最常見的「模板陷阱」，每一條都應明確禁止：

- **置中 Hero 大字 + 模糊全版背景圖**：幾乎所有 AI 生成的文學網站都從這裡開始。改用：篇名大字靠左對齊，圖片在頁面右側一半，或完全在文字後面才出現。
- **radius-8 或 radius-12 的白色卡片容器**：圓角白卡是 SaaS 產品的設計語言，不是散文集。去掉所有卡片框，圖片直接放在背景上。
- **System-blue `#007AFF` 或 Material-teal 作為強調色**：這些顏色和土地、水塘、竹林沒有任何關係。每篇的強調色必須從照片中取樣。
- **千篇一律的 `opacity: 0 → 1` fade-in（所有元素）**：fade-in 不是「互動設計」，只是把視覺資訊延遲呈現。如果要用 scroll reveal，只用在「一個段落裡最關鍵的那一句話」，不要全站上下全部 fade。
- **Google Maps embed、社群分享按鈕、浮動客服聊天氣泡**：任何「功能性 SaaS UI」元素都要清除乾淨。
- **Font Awesome icon 配文字按鈕**：用純文字超連結，或手刻的 SVG 線條，不要用圖示庫的通用 icon。
- **Gradient 漸層背景**（紫藍、粉橘）：AI 設計工具的預設美學。本案配色全部取自田野實景，不使用任何非自然的漸層。
- **「作者 Avatar 圓形照片 + 名字 + 一行介紹」的 author bio 卡片**：如果需要自我介紹，寫成一段完整的中文散文句子，放在關於頁，不要做成 UI 元件。
- **過度精緻的 loading animation**：純靜態 HTML/CSS/JS，載入速度快才是質感。不需要任何 loading screen。

## 技術實作備忘

- **字體**：`'Noto Serif TC'` 從 Google Fonts 載入（weight: 400, 700），標題 700 正文 400
- **行高（中文）**：正文 `line-height: 1.9`，深色背景版（魚篇）`line-height: 2.1`
- **正文欄寬**：`max-width: 42rem`（約 672px @ 16px base），`margin: 0 auto`
- **段落間距**：`margin-bottom: 2rem`，圖片上下 `margin: 3.5rem 0`
- **圖片全出血**：`.full-bleed { width: 100vw; margin-left: calc(-50vw + 50%); }`
- **Scroll reveal（限制用量）**：`IntersectionObserver`，threshold `0.3`，只加 `translateY(12px) → 0` 的微小位移，duration `0.6s ease`，不要 scale 或 rotate
- **章節跳轉導覽**：固定在左側的細豎線 + 三個小圓點（非 hamburger menu）——簡單、不佔空間
